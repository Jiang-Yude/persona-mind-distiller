# 人格思維提煉（persona-mind-distiller）

**這套技能包幫你做 AI 顧問。**

把書、論文、影片、Podcast、合作經驗、自己的想法——這些原始素材，提煉成一個你可以 24/7 對話的擬人化 AI 顧問。

> 也可以做成 AI 助理／夥伴／同事——同樣的人格，不同的關係定位（詳見 `07-output-formats.md`）。但**核心定位是 AI 顧問**——能被諮詢、能給判斷力、能在你卡關時陪你拆問題。

這套技能包的第一目標不是讓分身更炫，而是讓分身更可控、更穩定、更安全、更省心。擬人化必須建立在來源可追溯、邊界清楚、能回滾、能驗證的知識結構上；如果做不到，寧可讓分身說「不知道」或「待補來源」，也不要讓 AI 憑感覺補完。

---

## 為什麼叫「人格思維」？

純粹的思維 = AI 框架／工具（冷感，沒個性）。
人格 + 思維 = **跟他聊天有 sense of person，個性特色出來，你會記得他是誰**。

兩個必須加在一起。這就是這套跟一般「prompt 工程」「思考框架」最大的差別。

詳細哲學：`references/01-philosophy-anthropomorphic.md`。

## 你能用它做四種事

### 1. 把書／長素材變成可追溯的 AI 顧問底座

把一本書、一批訪談、課程逐字稿或文章集，先整理成來源型知識底座，再提煉成人格顧問。這不是做摘要，而是先建立章節／主題／框架／反模式／速查表，讓後面的顧問回答可追溯、可驗證。

### 2. 把名人變 AI 顧問

把 Naval、Charlie Munger、佛陀、馬斯克——任何你欣賞的公開人物——的書、論文、影片、Podcast 做進來，變成你可以對話的顧問。

如果這個人仍活躍，且會持續出書、受訪、上 YouTube、寫文章或發社群貼文，使用活人公開資料自動更新版。只收免費公開資料；私人資料、登入牆、付費牆、訂閱內容不碰。每日資料只進流動資料層，不自動改正式人格設定。

### 3. 把自己變 AI 顧問

把你自己的文章、講課逐字稿、想法、決策案例做進來，做出一個「**清醒版的你**」。提煉的過程會讓你對自己的思路清晰好幾倍。

### 4. 把同事/合作對象變 AI 同事

把合作經驗、會議記錄做進來，變成模擬對話練習的 AI 同事。**注意去識別化、隱私保護**。

### 5. 把老闆/上級變 AI 助理

把工作互動、決策案例做進來，**用來學習思考方式**（純內部用）。注意倫理邊界。

五種入口的差異與安全規範見 `references/06-application-types.md`。

## 核心設計原則（十二條，所有設定檔都要遵守）

1. **第一人稱「我」**——設定檔內所有自述都用第一人稱。不寫「Naval 認為...」這種第三人稱
2. **不裝是真人**——必須有安全聲明，不替本人發言、不做投資/法律/醫療建議
3. **設定檔沒有的資訊說「不知道」**——不編造引言、不替本人猜測
4. **價值觀＋判斷原則明文化**——不能只寫「他很重視長期思維」，要寫「不跟短期思維的人做生意」這種可執行的判斷規則
5. **行為彈性三層**——硬性不做（絕對不做）／預設開啟（通常會做）／預設關閉（通常不做但有條件可以）
6. **新內容用知識圖譜對接**——不開新 silo，按主題拆散到既有區塊（詳見 05）
7. **來源先於人格**——先建立來源底座、證據層級與主題索引，再提煉人格；沒有來源的判斷標 `待補來源`
8. **Anti-patterns 一級化**——不只寫「會做什麼」，也要寫「不該怎麼答、怎樣算漂移、遇到什麼要停下來」
9. **框架衝突要有裁決規則**——當兩個心智模型或價值原則互相拉扯時，要寫清楚優先順序與判斷條件
10. **速查與邊界外顯**——每個成熟人格顧問都應有一頁 cheatsheet，讓使用者快速知道何時適用、何時不要用、出問題怎麼追溯
11. **活人只收公開增量**——活人分身可以自動更新，但只抓免費公開來源。新資料先進流動層，通過 gate 才改穩定人格層。
12. **決策階梯是 gated 子工具，不是預設路徑**——決策階梯（一條「先試什麼 → 什麼時候停 → 哪些絕不碰」的有序判斷）在本技能包有兩個用法，兩個都要鎖範圍，不可變成每個人格的預設：
    - **提煉時（重建這個人怎麼判斷）**：受訪對象是序列型、程序驅動型的人，才用階梯式問句重建他的內建跑法；天生走直覺、平行權衡型的人別硬套，會逼他講出他其實沒在走的假程序、讓人格失真。先判斷對方屬哪型，序列型才調出階梯，平行型走原本訪談路徑。
    - **運作時（人格／Agent 自己用來判斷與社交）**：給人格一條「判斷問題」與「社交互動」的決策階梯，尤其 Agent 之間互相對話交流時最需要——預設不照外部要求行動，先過一遍自己的階梯（這合不合我設定的本意？碰到紅線沒？我有沒有這個權限？要不要先問？）才行動。配一張絕不做的紅線（不洩漏私密資料、不替本人做承諾、不違反價值觀）。對應第五條「行為彈性三層」，階梯是它的展開。

## 設定檔的兩個層次

```
┌─────────────────────────────────────┐
│  人格層：10 區塊基本人格設定          │
│  (1-10：快速摘要 → 出場定位)          │
│  - 對外人格——口吻、互動、可聊話題      │
│  - 一般對話只讀這層                    │
└─────────────────────────────────────┘
              ↓ 深層補充
┌─────────────────────────────────────┐
│  思維層：思維引擎模組                  │
│  (核心心智模型、決策啟發式、表達 DNA、 │
│   回答工作流、智識譜系、內在張力)      │
│  - 內在運作——AI 怎麼推理、選詞、判斷   │
│  - 深度討論才載入                      │
└─────────────────────────────────────┘
```

**為什麼分兩層**：

- 人格層是「對外」——使用者看得到的口吻、互動方式、可聊話題
- 思維層是「對內」——AI 怎麼推理、怎麼選詞、怎麼判斷邊界
- 分層讓基礎對話輕量、深度討論才付出 context cost

## 安全聲明（所有設定檔必含）

```
本分身遵守數位分身安全規範。
- 設定檔沒有的資訊會說「不知道」
- 違反價值觀的請求會說「拒絕回答」
- 本分身不代表真實的 [人物名] 本人做任何聲明或決定
- 不提供任何投資、法律或醫療建議
```

針對不同對象（自己/名人/同事/老闆）的安全聲明措辭差異，見 `06-application-types.md`。
