# 活人公開資料自動更新版

這份 reference 用於「對象仍活躍，而且會持續公開發表」的人格思維提煉。

典型來源包含：

- 新書與公開書摘
- YouTube 影片與公開字幕
- Podcast 與訪談
- 公開文章與部落格
- 公開社群貼文
- 官方公司或團隊公告
- 新聞報導與評論

只收免費公開資料。私人資料、登入牆、付費牆、會員限定、課程內部資料、電子報付費內容不碰。

## 核心架構

活人自動更新版分兩層：

1. 穩定人格層：`SKILL.md`、人格卡、心智模型、決策啟發式、表達 DNA、安全邊界
2. 流動資料層：每日新增、原始資料、週報、更新候選、錯誤紀錄

每日更新只進流動資料層。穩定人格層不能被單日資料直接覆寫。

## 建議資料夾結構

```text
person-live-skill/
├── SKILL.md
├── references/
│   ├── source-policy.md
│   ├── promotion-gate.md
│   ├── update-system.md
│   ├── source-config.json
│   ├── upstream/
│   └── live-data/
│       ├── raw/
│       ├── daily/
│       ├── weekly/
│       ├── review-queue/
│       ├── state/
│       └── logs/
└── scripts/
    └── update_live_data.py
```

如果使用 Codex App，排程放在 Codex App 內建「自動化」清單，不另建看不見的系統排程。這樣使用者能集中管理所有 Codex 做過的固定任務。

## 來源分級

至少分成四類：

- `person_direct`：本人直接發表，如公開貼文、公開訪談、公開演講、公開文章
- `organization_primary`：本人公司、團隊、基金會或出版社的一手資料
- `news_report`：媒體報導的可核對事實
- `news_commentary`：評論、社論、市場解讀、第三方分析

回答時要分清楚：

- 本人說了什麼
- 組織做了什麼
- 媒體報導了什麼
- 外界怎麼解讀

`news_commentary` 不能寫成本人立場。

## 自動更新節奏

- 每日：搜尋公開新資料，產 daily digest
- 每週：整理 weekly synthesis，找反覆出現的主題
- 每月或重大事件後：依 promotion gate 判斷是否更新穩定人格層

## Promotion gate

新資料要改正式人格設定，必須通過四關：

1. 來源可靠：本人直接來源、組織一手來源，或兩個以上可靠報導
2. 重複出現：不是單日情緒、玩笑、爭議或媒體噪音
3. 結構影響：會改變心智模型、決策啟發式、反模式、時間線或誠實邊界
4. 可替換：能指出應該改哪一段，以及舊段落哪裡不足

沒通過 gate 的材料只留在 daily 或 weekly。

## 公開社群貼文

優先順序：

1. 官方 API 或平台允許的匯出方式
2. 公開網頁 best-effort 抓取
3. 搜尋引擎索引結果
4. 新聞或第三方引用

公開網頁 best-effort 抓取只當備援。它可能因頁面結構、登入限制、反爬機制而失效。抓不到時不要硬繞，也不要要求使用者開瀏覽器交出登入狀態。

## 安全邊界

- 不冒充本人
- 不替本人做未公開聲明
- 不用單則貼文腦補人格
- 不碰私密、付費、登入牆資料
- 不自動發文、留言、按讚
- 不把新聞評論當成本人觀點

## 首例

`elon-musk-live` 是本模式首例：

- 每天 08:30 由 Codex App 內建自動化更新
- 先抓公開新聞、評論與公開 X 頁面可取得內容
- X API token 非必要；若日後設定，只作為更穩定來源
- 正式 `SKILL.md` 不每日自動改，只在通過 gate 後更新
